{% trans "IT Infrastructure teams are typically required to conduct, assist in, and endure a number of different types of audit every year - finance audits, security audits, risk-related audits, budget cycle audits, and more." %}

{% trans "A software audit can be particularly difficult if" %}:

  1. {% trans "You don’t have an accurate inventory of all your software" %}
  2. {% trans "The data you do have exists only in somebody’s head, or you have personnel changes" %}
  3. {% trans "Software is frequently installed and removed on servers and end-user machines without record" %}

{% trans "A common strategy is to attempt a “full inventory” in order to prepare for a software audit, with results stored in Excel and/or Visio files." %}

{% trans "This approach is problematic for several reasons" %}:

  1. {% trans "It can take weeks or months to get everything documented and organized into Excel. " %}
  2. {% trans "By the time the software inventory is completed, so many things have been changed that it’s again out of date. At best, it will be out of date come the next audit." %}
  3. {% trans "Even if you create processes to keep your spreadsheet up-to-date, you will inevitably end up with multiple versions, resulting in data issues as there is no centralized editing support and no standardized data template." %}
  4. {% trans "While it might be possible to keep some data up to date manually, other data such as IP usage is virtually impossible to track manually." %}
